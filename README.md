# plaster

`plaster` comes after `BRICs` and `mortar`. plaster has the following functions:

- **Plastering**: Compute and cache sequences, frames, and other statistics. Check for and flag inconsistencies.
- Handle both old linux time and new GPS time data.
- Index and cache all the captured data as AVLTrees for time synced frame finding.
- Colmap-based data intrinsic and extrinsic calibration and caching.
